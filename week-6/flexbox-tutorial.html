What is Flexbox?

Flexbox is a one-dimensional layout system that distributes space along a single axis—either rows or columns. It makes common UI patterns (nav bars, media objects, split layouts) much easier than floats or table hacks.

With a flex container, children become flex items that intelligently size, wrap, align, and stretch. You control their behavior with a small set of properties that work predictably across screen sizes.

Flex Containers

The “parent” becomes a flex container when you set display: flex (or inline-flex). This flips on a layout context where all direct children are flex items. Key properties: flex-direction, flex-wrap, gap, align-items, and justify-content.

In this page, .container is the flex parent and its direct children are the sidebar and the content area—two columns, one fixed and one flexible.

Flex Items

Flex items respond to flex (shorthand for grow, shrink, basis). For example, a fixed sidebar can use flex: 0 0 300px while the content uses flex: 1 1 auto to take remaining space and shrink as needed.

You can also reorder items with order and override alignment per item using align-self.

Alignment & Justify

justify-content positions items along the main axis (left-to-right in rows, top-to-bottom in columns), while align-items works along the cross axis. Use gap instead of margins for clean, consistent spacing between items.

For complex vertical centering, flexbox eliminates historical hacks and gives you reliable alignment with a single property.

Real-World Patterns

Common patterns include fixed sidebars with flexible articles, card rows that wrap, and responsive toolbars that space controls intelligently. By mixing flex-basis, growth, and shrink rules, you can model many production layouts with minimal CSS.

Remember: Flexbox is one-dimensional; for full grids, prefer CSS Grid—or combine both where each shines.
